Option A: Modal/Dialog for Configuration

When user selects Multiple Promotion or Action-Based campaigns, a modal appears to gather initial configuration before proceeding to the create view.

Multiple Promotion Campaigns Setup

Create 2-10 coordinated promotion campaigns

Pros

  • Quick setup without leaving context
  • Familiar pattern (common in web apps)
  • Easy to dismiss and return
  • Minimal navigation complexity

Cons

  • Limited space for complex configuration
  • May feel constraining for advanced options
  • Overlay can obscure context

Option B: Different Create Views

Each template type gets a specialized create view optimized for its workflow. Multiple campaigns get a tabbed or card-based interface.

Pros

  • Specialized UI for each workflow type
  • Clear visual separation between campaigns
  • Easy to switch between campaigns
  • Can optimize form fields per template

Cons

  • More code to maintain (multiple views)
  • Steeper learning curve (different UIs)
  • Takes longer to develop

Option C: Intermediate Configuration Step

After selecting a template, users go through a dedicated configuration screen before reaching the main create view.

Campaigns / Configure Multi-Campaign

Configure Multiple Promotion Campaigns

Set up your campaign structure before filling in details

Pros

  • Clear, focused configuration step
  • Sets up structure before details
  • Allows for template-specific questions
  • Progressive disclosure of complexity

Cons

  • Extra step in the flow
  • May feel like extra work for simple cases
  • Users can't skip ahead

Option D: Dynamic Create View

The create view adapts based on template selection, showing additional sections or modifying existing ones dynamically.
