Cross-Platform Template Creation

The "Build Template" flow may live on a separate platform with its own backend. This document explores two integration patterns (embedded iframe vs. redirect) and prototypes the redirect experience.

01 — Research
Option A
Embedded UI (iframe / microfrontend)
Load the other platform's template builder inside a modal or panel on this page. User never leaves — the external UI renders in-context.
+
Seamless UX — user stays in context, no page switch
+
Good if same SSO / auth — session can pass through
–
Requires iframe-friendliness — other platform must allow embedding (CSP, X-Frame-Options)
–
Complex auth plumbing — token sharing, CORS, postMessage security review needed
–
Harder to debug / test — cross-origin errors are opaque
!
Responsive issues — other platform's layout may not adapt to modal viewport
Best when: both platforms are on the same domain / SSO and the external team can commit to iframe support long-term. High eng cost upfront.
Option B · Recommended
Redirect with Context + Return URL
When user clicks "+ Create", pass campaign context as URL params and redirect to the other platform. After saving, the other platform redirects back with a template ID.
+
Simple to implement — just URL params + a return callback
+
No auth/CORS complexity — each platform handles its own session
+
New-tab option — user can preserve their current work while building
–
Context switch — different URL / app experience, potential for disorientation
–
Requires partner coordination — other platform must implement return URL + template ID response
!
If partner can't return callback — user can still manually link by copying a template ID
Best when: platforms have separate auth, or the external team isn't ready for iframe support. Lower eng cost, ships faster.
Dimension Embedded iframe Redirect + Return URL
Implementation effort High — security review, postMessage API, iframe sizing Low — URL params + listen for return redirect
Auth complexity High — must share session or tokens across origins Low — each platform handles its own auth
User experience Seamless — no context switch Context switch, mitigated by new-tab option + clear navigation
Dependencies on partner Must allow iframe embedding (CSP headers) Must implement return URL callback (or fall back to manual template ID)
Graceful degradation iframe blocked = broken, no fallback Can fall back to "copy template ID" if callback not available
Recommended for Same domain / SSO + eng capacity Different backends · near-term
Recommendation: Start with the redirect pattern. It ships faster and doesn't require iframe coordination with the other team. If the other platform can implement a return URL callback, the UX is clean — user opens in new tab, builds the template, and returns to find it auto-linked. The embedded approach can be revisited as a V2 if the teams align on auth.

03 — Fallback: Manual Template ID

If Creative Studio can't implement the return URL callback, users can still link templates manually by pasting a template ID. This is the lowest-dependency fallback — no partner eng work required.
